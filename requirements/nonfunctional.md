## Task 1.2 Non-Functional Requirements (User Stories)

1. Responsive UI - As a user, I want the platform to be responsive so that I can use it comfortably on different devices/screen sizes.

2. Lightweight performance - As a user, I want the platform to be lightweight so that it loads and runs fast without unnecessary overhead.

3. Usability / intuitive - As a user, I want the platform to be intuitive to use so that I can complete tasks like searching and ordering without confusion.

4. Maintainability & extensibility - As a developer/maintainer, I want the system to be easy to maintain and extend with new meal types or dietary rules so that we can add features with minimal rework.
